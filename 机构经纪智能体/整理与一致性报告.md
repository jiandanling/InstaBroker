# 机构经纪智能体 - 整理与一致性报告

## 一、已完成的整理

### 1. 内联样式清理
- **员工-办理任务.html**：流程图 SVG 的 `style="max-height: 200px;"` 已改为使用公共类 `.flow-diagram-svg`（定义在 `css/common.css`）。
- **index.html**：主内容区 `style="scrollbar-gutter: stable"` 已改为使用公共类 `.scrollbar-gutter-stable`（定义在 `css/common.css`）。

### 2. 公共样式补充（common.css）
- `.flow-diagram-svg`：流程图 SVG 最大高度 200px，供办理任务页复用。
- `.scrollbar-gutter-stable`：主壳滚动区域预留滚动条空间，避免布局抖动。

### 3. 当前状态
- 所有子页均已引入 `css/common.css` 与 `js/spa-wrapper.js`，并调用 `ensureWrappedInIndex(文件名)`；**index.html** 为主壳，不引入 spa-wrapper。
- 各 HTML 中**无**页面内 `<style>` 块，**无**剩余内联 `style` 属性。

---

## 二、风格一致性分析

### 1. 结构规范（一致）
- 子页均为：Tailwind CDN → common.css → spa-wrapper.js → 内联脚本调用 `ensureWrappedInIndex`。
- 子页主体内容均包在 `<main class="subpage-main">` 内，无独立 header/footer，符合 SPA 注入约定。
- 对话/消息类页面（如员工-风险处理、员工-拜访任务、员工-业务引入）均使用 `visit-page-root`、`message-list-column`、`section.min-w-0` 等统一结构。

### 2. 可统一的小差异（建议后续按需处理）

| 项目 | 说明 |
|------|------|
| **DOCTYPE/格式** | 部分为 `<!DOCTYPE html>`，部分为 `<!doctype html>`；部分 meta/link 自闭合 `/>`，部分未自闭合。仅格式差异，不影响功能。 |
| **缩进** | 有的子页 2 空格，有的 4 空格，可择一统一。 |
| **文字颜色** | 多处使用 Tailwind 任意值 `text-[#1E293B]`、`text-[#64748B]` 等，与 common.css 中的 `--color-text`、`--color-text-muted` 含义一致。可选：在 common 中增加 `.text-primary`、`.text-muted` 等工具类，逐步替换为统一类名，便于日后改主题。 |
| **卡片样式** | 子页多用 `subpage-card-shadow`、`rounded-xl`、`bg-white`，与 common 的卡片体系一致；个别仍写 `shadow-md` 等，可逐步统一为 common 的 card 类。 |

### 3. 与其它目录页面的关系
- 本报告仅针对 **机构经纪智能体** 目录下 HTML。若项目其它目录（如根目录或其它模块）有独立页面，其风格是否与“主壳 + common.css + 子页注入”一致，需在对应目录下单独对照检查。

---

## 三、建议后续可做（可选）

1. **统一 DOCTYPE 与缩进**：在 机构经纪智能体 内统一为 `<!DOCTYPE html>`、meta/link 自闭合、2 或 4 空格缩进其一。
2. **颜色类统一**：在 common.css 增加基于 CSS 变量的工具类（如 `.text-primary`、`.text-muted`），逐步替换各页中的 `text-[#1E293B]`、`text-[#64748B]`。
3. **与其它模块对比**：若存在其它入口或模块页，做一次“主壳 + 公共样式 + 子页结构”的横向对比，保证整站风格一致。

以上为本次整理与一致性结论；功能与现有风格均已保持，仅移除内联样式并抽成公共类。
