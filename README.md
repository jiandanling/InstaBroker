# 机构经纪智能体

机构经纪超级数字员工与组织者智能体演示项目。纯前端静态页面，面向机构经纪业务场景，包含员工端和组织者端两类入口。

## 产品简介（领导汇报版）

### 一、产品目标

针对机构经纪业务中信息传递链条长、协同效率低等痛点，本产品构建超级数字员工与组织者智能体双端协同体系，实现从前端展业到后端协同的全流程数智化支撑。

### 二、核心能力

产品采用员工端与组织者端双端设计：

- **员工端（超级数字员工）**：承接商机发现、客户拜访、方案沟通、业务办理等一线工作，自动接收公司自动驾驶大模型发现的商机及组织者下发的任务。
- **组织者端（组织者智能体）**：负责协同评估、风险处置、业绩分析、应急组织等中台能力，对复杂业务进行研判并协调多部门资源。
- **圆桌会议机制**：涉及多部门协同的业务，自动触发圆桌会议，各方共同研判并形成处置意见，实现跨端、跨部门高效协同。

### 三、典型流程

- **正向展业闭环**：商机自动推送至员工端，附带拜访材料；客户经理完成拜访并上传纪要，公司自动驾驶大模型自动更新客户档案；组织者生成方案并下发至员工端，客户经理二次拜访确认；客户认可后下发开户任务，公司自动驾驶大模型提供办理指引。
- **风险协同闭环**：公司自动驾驶大模型监测到风险事件后推送至员工端；客户经理一键跳转至组织者端；超级数字员工向公司自动驾驶大模型查询客户交易情况；向组织者智能体请求发起圆桌会议，协同多部门形成处置方案并下发执行。
- **多智能体协同**：员工端提供圆桌会议入口，展示待办会议列表；选择会议后跳转至组织者端参与研判；涉及多组织者的复杂业务，系统自动触发协同评估，形成综合结论后下发执行。
- **日常协同**：组织者端集中管理请示事项、应急组织、经营数据分析等，支撑管理决策。

### 四、产品特点

- **双端协同**：员工端与组织者端打通，实现端到端流转，避免信息孤岛。
- **智能驱动**：商机、任务、风险由公司自动驾驶大模型或智能体自动发现与下发，降低人工成本，提升响应速度。
- **多方协同**：复杂业务涉及多部门智能体联合评估，形成统一决策的协同模式。

## 技术栈

- HTML5 + Tailwind CSS (CDN)
- 纯前端，无后端依赖，无构建工具
- 可直接用浏览器或简单 HTTP 服务打开

## 导航关系

- **员工端**：主入口 `index.html`，各员工页返回 `index.html`
- **组织者端**：主入口 `组织者-index.html`，多数组织者页返回 `组织者-index.html`
- **跨端**：`员工-圆桌会议.html` 可跳转至组织者综合评估、风险处理等页面；`index.html` 中博达富通风险卡片可跳转至 `组织者-博达富通-风险处理.html`
